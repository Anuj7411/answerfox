## Phase 2 scope (next chunk)

**Branch**: `feat/v0.5-coverage-parity-phase-2`

### 5 new audit checks

1. **A12 RFC 8288 Link headers** — needs crawler refactor (add `headers: Headers` to `FetchAndParseResult`, expose in `AuditDom`). Check looks for `Link:` header with rel attributes. **1 pt, low severity, meta-and-technical**.

2. **A13 AI bot rules in robots.txt** — fetch `/robots.txt`, look for explicit `User-agent: GPTBot`, `ClaudeBot`, `PerplexityBot`, `Google-Extended`, etc. Pass if ≥1 explicit AI bot rule, warn if only generic `*`, fail if all AI bots blocked. **2 pts, medium, meta-and-technical**.

3. **C3 Markdown content negotiation** — fetch the audited URL with `Accept: text/markdown`. Pass if returns markdown (content-type or body shape), fail if HTML only. **2 pts, low, structured-data**.

4. **C4 Content Signals** — check for `X-Content-Signals` HTTP header or `<meta name="content-signals">` tag. Pass if present + has citation/usage policy field, warn if present but minimal. **2 pts, medium, structured-data**.

5. **G8 Web Bot Auth (HTTP Message Signatures, RFC 9421)** — check if origin responds with `Signature-Input` and `Signature` headers. Pass if signed correctly, fail if not. **4 pts, low, agent-readiness**.
